# Release 运动测试日志导出设计

> 状态：已落地，并于 2026-07-18 通过独立六维代码与自动化复验；Release/Play 真机导出和小朋友实际动作仍按测试手册验收。

## 背景

当前识别追踪只在 Debug 构建中启用。它已经按训练会话写入 JSONL，包含逐帧关键点、准备态与运动门控、相对深度、计数器状态、处理耗时和丢帧数，但商店 Release 包无法开启，也没有 App 内导出入口。真实场景问题往往在日常使用中偶发，因此需要让用户主动开启本地诊断记录，并在事后导出给电脑分析。

## 已确认决策

- 商店 Release 包提供“运动测试日志”设置，默认关闭，用户明确开启后才记录。
- 设置从下一次进入训练时生效；关闭设置不会删除已经保存的日志。
- 日志仅存放在 App 私有目录，不上传；不记录照片、视频和音频。
- 最多保留最近 20 次训练，同时限制单次 12 MiB、总量 24 MiB；超过数量或总量后按时间删除最旧会话，单次触顶后写入截断事件并停止。
- 导出采用方案 A：通过 Android 系统文件选择器将最近日志汇总为一个 `.jsonl` 文件，用户可保存到“下载”等目录，再通过 USB 复制到电脑。
- 导出文件第一行是 `export_manifest`，至少包含格式版本、导出 UTC 时间、App 版本/构建号、会话数量和 `containsRawMedia: false`；其后按会话时间顺序写入边界记录和原始 JSONL。
- 没有日志时明确提示；用户取消保存不视为错误；记录与导出失败都不得影响训练。
- 本功能不修改识别信号、阈值、门控或计数状态机。
- 真机设置页必须让开关状态一眼可辨：标题旁显示本地化的开启/关闭标签；关闭态使用高对比轨道、边框和叉号，开启态使用主绿色轨道、勾号和状态标签。窄屏时标题与标签允许换行，不自定义新的交互组件。

## 分层与数据流

1. `AppSettingsStore` 持久化布尔开关，`AppSettingsController` 暴露并通知 UI。
2. `HomePage` 在打开训练页时读取当前开关；`WorkoutPage` 用该值构造 `RecognitionTraceLog` 并注入 `WorkoutController`。这样开关只在训练会话边界生效，不在逐帧热路径读取存储。
3. `RecognitionTraceLog` 继续负责私有目录写入和会话轮转，默认留存量从 10 调整为 20。
4. 新增基础设施层 `RecognitionTraceExportService`：读取已关闭的会话文件、构造汇总 JSONL，并调用项目已有 `file_picker` 的 `saveFile(bytes: ...)` 打开系统保存界面。
5. 个人页设置弹层提供开关和导出入口；所有用户文案进入中英文 ARB。

## 安全与边界

- Release 默认关闭；Secure Storage 中不存在该键或读取失败时均回落为关闭。
- 训练中不会出现设置页导出入口；正常训练停止后日志 sink 已 flush/close，再进行汇总。
- 导出仅复制日志，不删除源文件；关闭开关也保留源文件，便于问题发生后补导出。
- 私有日志随 App 卸载而清除；只有用户在系统保存界面确认位置后，副本才离开私有目录。
- 日志 I/O 保持 best-effort，异常不得改变相机、推理、计数、语音或训练保存流程。
- 不改 `WorkoutController` 的异步 session 守卫。
- 活跃会话只写 `.jsonl.part`，正常关闭后才改名为 `.jsonl`；导出忽略异常退出的临时文件，并校验每个最终文件是逐行完整 JSON 对象。
- 汇总导出上限为 25 MiB，预检超限时不把日志读入内存，也不打开系统保存界面。
- 开关写入串行执行；持久化失败时回滚可见值并提示用户，避免重启后状态反转。

- Widget 测试覆盖设置页开关、开关保存失败反馈、隐私说明、导出成功/无日志/超限/失败反馈及英文文案。
