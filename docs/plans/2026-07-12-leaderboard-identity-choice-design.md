# 运动广场身份选择设计

日期：2026-07-12
状态：已确认

## 目标

用户加入运动广场时主动选择公开身份，在展示社区成员存在感的同时，确保 Google 或 App 个人资料不会未经同意出现在榜单。

## 身份模式

### 使用当前个人资料

- 优先使用用户在 App 内设置的昵称和头像。
- 未设置 App 昵称或头像时，分别回退到 Google 名字或 Google 头像。
- 加入后继续跟随个人资料变化，榜单自动显示最新内容。
- 只公开用户确认过的名字和头像，不公开邮箱。

### 榜单专用身份

- 用户单独设置榜单昵称和预设头像，不影响个人资料页。
- 昵称必须唯一；冲突时提示用户更换。
- 昵称和头像加入后均可随时修改，不设置冷却期。

### 匿名参加

- 统一显示“匿名训练者”。
- 系统为每个匿名用户分配一个稳定的彩色匿名头像，用于区分榜单成员。
- 不返回 Google 名字、Google 头像、App 昵称或 App 头像。
- 加入界面默认选中匿名模式。

## 加入与修改流程

用户点击“加入广场”后，从底部弹出“选择你在运动广场中的身份”，展示三张可选择卡片和实时预览：

1. 使用当前个人资料，并说明资料变化后榜单会自动更新。
2. 设置榜单专用身份；选中后展开昵称输入框和现有预设头像选择区。
3. 匿名参加，并说明不会公开个人资料。

底部只提供“取消”和“确认加入”。提交失败时保留选择和输入内容，允许直接重试。

加入后，在“我的排名”卡片提供轻量编辑入口。用户无需退出榜单即可切换身份模式，或修改榜单专用昵称和头像。切换为匿名后，服务端立即停止返回此前公开的资料。

## 数据与服务端规则

`leaderboard_profiles` 保存：

- 身份模式：`profile`、`custom` 或 `anonymous`。
- 榜单专用昵称及用于唯一校验的规范化昵称。
- 榜单专用预设头像。
- 稳定的匿名头像键。

旧数据没有身份选择时一律按匿名处理，不能在升级后自动公开个人资料。榜单专用昵称通过数据库唯一约束处理并发抢占。

排行榜查询由服务端根据身份模式解析最终允许公开的显示名、预设头像或网络头像；Flutter 只渲染服务端结果，不在客户端重复实现隐私判断。

## API 与兼容

- 扩展加入接口，使新版 App 同时提交身份选择。
- 旧版 App 仍按原格式调用加入接口时，服务端默认匿名，保持兼容。
- 增加已加入用户修改榜单身份的接口。
- 排行榜响应继续保留现有排名与统计字段，并增加最终公开显示所需的网络头像字段；旧版 App 会忽略新增字段。
- Worker 先兼容部署，再安装新版 App 真机验收。

## 错误与降级

- 专用昵称非法或重复：弹窗保持打开并显示明确错误。
- 网络或服务端失败：保留用户输入，允许重试，不误报加入成功。
- Google 网络头像加载失败：显示安全的默认头像。
- 当前个人资料缺少某个字段：该字段单独回退到 Google 信息；Google 信息也缺失时使用安全默认值。
- 匿名头像由服务端持久保存，刷新、换手机或重新安装后保持稳定。

## 验证

Worker 真实 SQLite/D1 测试覆盖：

- 三种身份模式的公开字段。
- 旧加入请求和旧数据默认匿名。
- 榜单专用昵称唯一、修改和模式切换。
- 当前个人资料变化后自动反映到榜单。
- 日榜、周榜、0 次成员、退出和过期会员规则保持不变。

Flutter 测试覆盖：

- 三选一加入弹窗、默认匿名和实时预览。
- 专用昵称输入、头像选择、错误保留和重试。
- 已加入用户修改身份。
- 预设头像、Google 网络头像及加载失败兜底。
- 中英文文案和基本无障碍语义。

## 不包含

- 不公开邮箱。
- 不上传自定义图片；榜单专用头像继续使用现有预设图标。
- 不增加关注、评论、点赞或用户主页。
- 不改变训练统计、会员资格或排行榜计分规则。
