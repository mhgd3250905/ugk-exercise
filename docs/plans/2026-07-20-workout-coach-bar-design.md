# 训练教练状态栏自适应设计

日期：2026-07-20

## 目标

训练页唯一的教练状态栏根据当前本地化文案连续调整宽度：短文案不再占满相机舞台，长文案仍以当前截图中的宽度为上限；同时用绿色背景明确表达训练已经进入可开始或运行阶段。

## 视觉规则

- 状态栏最小宽度为 `150dp`，避免极短中文文案形成过窄胶囊。
- 状态栏最大宽度保持为屏幕宽度减去左右各 `24dp`，长英文最多显示两行。
- 文案、圆点、间距和水平内边距共同决定中间宽度，不按字符数设置离散档位。
- 保留两行文案的固定高度，避免准备姿势提示在防抖切换时带动计数控制台上下跳动。
- 宽度和颜色变化使用短时动画；系统要求减少动态效果时由 Flutter 的现有动画策略自然收束，不新增业务状态。

## 状态颜色

- `WorkoutStatus.readyToStart` 与 `WorkoutStatus.training` 使用主题主绿色背景和 `onPrimary` 前景色。
- 加载、启动相机、姿势引导、保存和错误等其他状态继续使用现有中性抬升表面。
- 绿色状态下圆点使用同一高对比前景色，避免圆点淹没在绿色背景里。

## 实现边界

- 只修改 `WorkoutPage` 的展示参数和 `_WorkoutCoachBar`，不改变 `WorkoutController`、识别状态机、计数或语音。
- 页面将当前 `WorkoutStatus` 作为语义状态传给状态栏；组件不根据本地化字符串猜测状态。
- 不新增颜色常量或第三方依赖，直接使用现有 `ColorScheme` 与页面表面色。

## 验证

- Widget 测试证明短英文状态明显窄于长英文状态，长状态不超过页面上限且不溢出。
- Widget 测试证明 `readyToStart`、`training` 使用绿色背景，普通引导状态保持中性背景。
- 既有窄距提示 300ms 防抖和固定高度测试继续通过。
- 完成后运行 `flutter analyze`、`flutter test`、回放基线及 `git diff --check`，再在已安装 Debug 真机上烟测状态变化。
