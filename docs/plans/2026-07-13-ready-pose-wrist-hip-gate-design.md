# 准备态腕髋门控设计

## 目标

只收紧进入准备态前的姿势判断，拒绝“跪坐、上身直立、双手自然下垂”这类并未手撑地的姿势。进入准备态后的计数链路保持不变，包括：

- 准备态头肩到地面的 50% 相对下压门槛；
- 头肩轨迹计数；
- 手臂离屏容忍；
- 回升计数与肘角反证。

## 真机证据

两次 Debug 训练均只进入一次准备态，分别正确记录 20 次和 15 次。第二次在坐立垂手时提前进入准备态：

- 鼻、双肩、双腕、双髋置信度均通过 0.3；
- 双腕低于双肩约 177–189px；
- 头肩中心稳定 514ms，通过现有 500ms 门槛；
- 但双腕与髋部几乎同高，符合自然垂手，而不是手撑地。

现有规则只要求腕低于肩，髋只参与置信度检查，因此该姿势按现行定义必然通过。后续姿势转换产生的一次候选动作被肘角反证拒绝，所以本次没有多计数；这不能替代正确的准备态判断。

## 方案比较

### 方案 A：腕低于髋固定像素

代码最少，但拍摄远近变化时含义不同，会重复固定像素阈值的问题，不采用。

### 方案 B：腕髋距离相对同侧躯干高度（采用）

对左右两侧分别计算：

```text
腕髋距离 = wristY - hipY
躯干高度 = hipY - shoulderY
要求：腕髋距离 >= 0.3 × 躯干高度
```

两侧都必须满足，不平均左右腕。该比例随人物画面尺度一起变化，并复用准备态已经强制可见的肩、髋、腕。

当前真机样本中：

- 有效支撑姿势两侧比例约为 0.90 和 1.00；
- 坐立垂手姿势两侧比例约为 0.14 和 -0.02。

`0.3` 位于两类样本之间，作为首轮实验值；后续只依据新增真机负样本调整。

### 方案 C：增加膝或踝的地面线

区分能力更强，但近距离准备时膝踝可能出框，会破坏现有近景使用场景，不采用。

## 数据流与边界

新条件只加入 `ReadyPoseGate.isPoseVisible`：

```text
未 ready
  → 核心点置信度
  → 双腕低于双肩
  → 双腕分别明显低于同侧髋部（新增）
  → 居中并稳定 500ms
  → ready
```

明确不修改：

- `motionPoseUsable`；
- `WristAnchor`；
- `PushupPipeline`；
- `PushupCounter`；
- `readyDepthRatio = 0.5`。

若腕髋条件失败，沿用现有行为：本帧不进入准备态，并重置稳定计时。无需新增状态、日志格式或 UI 文案；现有 Debug 关键点日志已经足够追溯。

## 验证标准

1. 双腕低于肩、但停在髋部附近的坐立垂手姿势不能进入准备态。
2. 标准支撑姿势仍可在稳定 500ms 后进入准备态。
3. 左右任一侧不满足即拒绝，禁止平均两侧结果。
4. 近景和远景的等比例姿势判定一致。
5. 现有计数测试、快动作测试、手臂离屏测试不改行为。
6. 全量测试通过，回放基线保持 step0=5 / v3=5 / v4=3。
