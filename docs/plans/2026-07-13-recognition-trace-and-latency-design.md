# 识别追踪与计数延迟优化设计

## 目标

- 消除计数管线中的重复平滑，缩短完成俯卧撑到计数/语音触发的等待，并降低快动作漏计概率。
- 在 Debug 包中持久记录每次训练的识别输入、门控状态、计数状态与耗时，便于真机问题事后追溯。
- 保持回放基线 step0=5、v3=5、v4=3 不变。

## 方案

`PushupCounter` 已对核心位移信号做 5 帧中值滤波。`PushupPipeline` 不再额外执行 5 帧移动平均，避免同一信号连续经过两层滞后滤波。中值滤波继续承担单帧毛刺抑制。

新增基础设施层 `RecognitionTraceLog`：

- 仅 Debug 模式由 `WorkoutController` 启用；Release 不写日志。
- 每次训练写一个 JSONL 文件到应用私有目录 `files/recognition_traces/`。
- 每个已处理帧记录关键点、准备门控、手腕稳定性、算法信号、计数器状态、处理耗时和丢帧数；关键状态变化另写事件记录。
- 自动保留最近 10 个训练文件；写入失败不得影响训练。
- 日志只保存在本机，不上传、不纳入 Git。Debug 包连接电脑后使用 `adb shell run-as <package>` 查看或导出。

## 验证

1. 先新增真实 `PushupPipeline` 快动作测试：稳定高位后，3 帧下压 + 3 帧还原应计数一次。
2. 为日志新增写入、禁用和轮转测试。
3. 运行 `flutter analyze`、完整 `flutter test`，确认回放基线仍为 5/5/3。
4. 构建并安装 Debug APK，在真机完成训练后通过 ADB 验证 JSONL 可读取。
