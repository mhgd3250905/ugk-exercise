# 准备态相对下压深度设计

## 目标

用准备态里已经可靠可见的头肩和双腕，为本次训练建立画面尺度；近距离需要更大的像素下压量，远距离允许更小的像素下压量，避免固定像素阈值造成近距离误计和远距离漏计。

## 已确认规则

准备态通过时保存：

- `readyTopY`：鼻子与双肩组成的初始头肩高度。
- 左、右手腕各自的地面高度；不平均两个独立手腕。
- `groundSpan`：取两个“头肩到对应手腕”的高度距离中较大的一个，作为保守尺度。

相对下压深度为：

```text
(当前头肩高度 - readyTopY) / groundSpan
```

只有相对深度达到 `50%`，计数器才允许从 up 进入 down。进入 down 后继续使用现有的中值滤波、最低点追踪、回升滞回和肘角反证，不要求运动态手腕继续可见。

## 为什么先采用混合方案

- 不完全替换现有计数器，避免重新引入快动作和回升不足漏计。
- 不要求严格回到准备态顶部，因为真机日志显示正常动作的顶部会随训练产生一定漂移。
- 固定像素值只保留为 MoveNet 抖动的最低噪声保护；主要深度标准改为准备态相对尺度。

两组 Debug 日志中，疑似误计动作达到准备态尺度的约 `45%`；其余动作范围为 `57%–77%`。因此 `50%` 是实验起点，后续仍以真机日志校准。

## 生命周期与异常

- 新训练、切换相机、lost-pose 退出 ready 时清除标定。
- 每次重新进入 ready 都重新标定。
- 标定数据无效时不得进入正式计数。
- Debug 日志记录准备态尺度、所需下压位置和逐帧相对深度。

## 验证

- 45% 的浅调整不计数。
- 50% 以上的下压回升计数。
- 不同人物画面尺度下，相同比例动作行为一致。
- ready 后手臂离屏仍可计数。
- 完整测试通过，回放基线保持 step0=5 / v3=5 / v4=3。
