# 排行榜长按用户操作设计

日期：2026-07-15

## 目标

移除排行榜用户卡片右侧的更多菜单图标，恢复昵称、名次与次数的整齐对齐；将低频的举报和屏蔽操作收进长按手势，降低视觉干扰与误触。

## 已确认方案

- 仅其他用户的卡片响应长按；当前用户自己的卡片不提供审核操作。
- 长按先提供轻微触觉反馈，再打开遵循浅/深色主题的底部操作面板。
- 面板顶部显示目标用户昵称，操作区复用现有“举报头像”“举报用户”“屏蔽用户”能力。
- 举报使用中性图标；屏蔽使用警示色并保留原有二次确认。
- 举报原因选中后立即提交，但提交期间必须显示加载反馈；成功提示“已举报并屏蔽该用户”，失败保留现有可重试语义。
- 卡片提供无障碍长按语义，不在榜单增加永久提示文案。

## 边界

本次只修改 Flutter 排行榜 UI、l10n、Widget 测试和维护文档。`LeaderboardController`、API client、Worker、D1 合同均保持不变；不新增依赖或图片资源。

## 验收标准

1. 用户卡片右侧不再显示菜单图标，次数区域对齐。
2. 长按其他用户卡片能打开主题化操作面板；长按自己的卡片无动作。
3. 举报、屏蔽和失败重试继续复用现有链路。
4. 举报提交过程有加载反馈，成功后有明确提示。
5. 中英文、浅深色无溢出，`flutter analyze` 与全量 `flutter test` 通过。

## 验证结果

- 本会话运行 `flutter analyze`：0 issue；`flutter test`：`390/390` 通过。
- 使用本机 production 配置构建 Debug APK，并通过 `adb install -r` 覆盖安装到模拟器，未卸载或清除 App 数据。
- 用户在模拟器人工确认：卡片末端对齐、长按震动、主题底部操作面板、取消操作及浅深色显示均正常。
- 本次 UI 人工验收未再次提交举报或屏蔽，没有新增 D1 测试写入。
