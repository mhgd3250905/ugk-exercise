# 训练历史本地缓存需求设计

日期：2026-07-20

## 背景

当前训练记录页只把 `GET /workouts?month=...` 返回的云端历史保存在本次页面的 `Future` 中。页面先读取本地训练文件，因此云端独有记录在每次进入时都会先消失，等待网络返回后才重新出现在日历里。云端返回没有写入持久化存储，App 重启、页面重进、断网或会员过期后都无法复用已经下载过的数据。

## 用户目标

1. 同一账号曾经成功下载的云端训练历史必须保存在本机。
2. 再次进入训练记录页时，先显示本地记录与已缓存云端记录，不出现“空日历 → 历史记录晚到”的闪烁。
3. 当前会员有效时，页面继续在后台请求当月云端历史；请求成功后补充缓存并刷新当前页面。
4. 当前会员失效时，不再请求、刷新或同步新的云端历史，但同一账号此前已缓存的记录仍可离线查看。
5. 云端请求失败时保留并显示旧缓存，不清空已有记录。
6. 缓存按 `appUserId` 隔离。退出登录或切换账号后，不得向无账号或其他账号展示原账号的云端历史；同一账号以后重新登录时仍可复用其缓存。

## 产品行为矩阵

| 账号/会员状态 | 首屏展示 | 云端请求 | 请求完成后的行为 |
|---|---|---|---|
| 已登录会员，已有缓存 | 本地记录 + 同账号缓存 | 请求当前月份 | 成功时合并并持久化；失败时保留旧缓存 |
| 已登录会员，无缓存 | 本地记录 | 请求当前月份 | 成功时显示并持久化；失败时只显示本地记录 |
| 已登录非会员，已有缓存 | 本地记录 + 同账号缓存 | 不请求 | 保持只读缓存，不清除 |
| 已登录非会员，无缓存 | 本地记录 | 不请求 | 不变 |
| 未登录 | 仅 ownerless 本地记录 | 不请求 | 不读取任何账号缓存 |
| 切换到另一账号 | 新账号本地记录 + 新账号缓存 | 仅新账号为会员时请求 | 旧账号数据保持隐藏 |

本次不新增“会员已过期”提示条或手动刷新按钮。会员有效时沿用现有“云端记录加载中/已合并/暂不可用”状态；会员失效时页面只显示可用数据，不显示云端刷新状态。

## 推荐架构

复用现有 `WorkoutSessionStore`，不新建第二份缓存文件。云端历史与本地训练具有相同的 `WorkoutSession` 数据合同，现有文件已经支持 `ownerAppUserId`、`syncStatus`、原子串行写入、账号过滤和按 ID 合并。

新增 `cacheCloudHistoryForOwner(ownerAppUserId, sessions)`：

- 把云端记录绑定到发起请求时捕获的账号。
- 以 `(ownerAppUserId, session.id)` 判断是否已存在，重复刷新保持幂等。
- 新增记录统一保存为 `WorkoutSyncStatus.synced`，避免它们被重新排入上传队列。
- 同账号已有本地/待同步记录时保留本地版本，不用云端副本覆盖本机的同步状态。
- 不覆盖其他账号即使 ID 相同的记录。
- 只有列表产生新增记录时才落盘，避免每次进页面重复写同一文件。

`HomePage` 在创建记录页时继续捕获当前 `SavedAccountSession`。只有 `AccountController.premium` 为真时才调用现有 loader；loader 成功后先把结果写入同一 `_store`，再把结果交给 `RecordsPage` 与首屏本地数据合并。非会员路径不创建网络 Future，但 `RecordsPage.loadForOwner(appUserId)` 会自然读到此前缓存。

## 数据一致性与竞态

- 缓存归属使用请求开始时捕获的 `appUserId`，不能在请求结束后重新读取“当前账号”再决定 owner。
- 云端请求晚到且期间账号已切换时，结果最多写入原账号自己的缓存，不得写入新账号；当前记录页也仍绑定打开页面时的 owner。
- 缓存写入失败不能隐藏已经成功返回的云端结果；当前页面仍可显示本次响应，但下次进入可能需要重新下载。现有云端不可用文案只表示请求失败，不把本地持久化失败误报为会员失效。
- 服务端训练记录视为不可变事实。本次只补充缺失记录，不设计云端删除、修改、分页、缓存过期或容量淘汰。

## 非目标

- 不改变 Worker、D1、`GET /workouts` 响应或会员授权合同。
- 不允许非会员发起云端读取、上传、恢复或刷新。
- 不扩大为按月切换时自动下载每个历史月份；本次只缓存现有代码已经请求成功的月份。
- 不删除用户现有本地记录，不在退出登录时清空训练文件。
- 不新增第三方依赖、数据库或安全存储结构。

## 验收标准

1. 云端独有记录成功下载后，在新建 `WorkoutSessionStore` 实例时仍能从同一文件读取。
2. 同一云端列表重复缓存不产生重复记录，也不覆盖同 ID 的本地待同步记录。
3. 两个账号即使出现相同 session ID，仍能分别读取自己的记录。
4. 会员重新进入记录页时，缓存数据在云端 Future 完成前已经显示。
5. 同一账号会员过期后进入记录页，缓存数据仍显示且 cloud loader 调用次数为 0。
6. 未登录或切换账号时不显示其他账号缓存。
7. `flutter analyze` 无 issue，`flutter test` 全绿，回放硬基线保持 step0=5 / v3=5 / v4=3，`git diff --check` 通过。
