> 这段是固定的开场白；具体功能需求在后面追加。

```

2. 遵守 $ponytail 规范
3. 设计过程排查勘误采用对抗式审查方式

关键约束（违反会埋坑，AGENTS.md 有详细说明）：

---

## 使用说明

1. 开新会话时，复制上面 ``` 内的全部内容
2. 粘贴发给 agent
3. 在后面追加本次的具体需求（如"我要加一个深蹲模式"）
4. agent 会先读 AGENTS.md 进入状态，再处理你的需求

## 为什么这样设计

- **固定部分**（最高声明 + 接手指引 + 约束）：每次都一样，保证 agent 行为一致
- **不重复 AGENTS.md 内容**：只引导它去读，避免两处维护、信息漂移
- **声明工作原则**（第一性 / ponytail / 对抗式审查）：让 agent 知道这个项目的开发文化
- **放仓库里**：版本管理，不会丢；改了模板所有人都看得到
