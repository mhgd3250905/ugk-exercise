## 发现清单

### 🟡 中等

#### M1. `switchCamera()` 的 session 检查晚于多个 await

**证据**：`lib/control/workout_controller.dart:146-173`

`switchCamera()` 在 `endOfFrame`、subscription cancel、等待帧管线空闲和 camera dispose 之后，才第一次检查 `session != _session`。这不符合项目文档“异步流程每个 await 后检查 session”的纪律。

`_switchingCamera` 已阻止快速连续相机切换，因此原报告描述的“连续切换互相交错”不准确。剩余风险是 stop/dispose 在 await 期间使旧切换失效后，旧流程仍继续执行资源操作；现有代码尚无直接覆盖该时序的 controller 测试。

**建议**：先用 fake camera/subscription 写一个“切换等待期间 stop/dispose”的竞态测试，再在会继续初始化或写状态的 await 后添加 session 守卫。不要只机械补两处检查。

---

#### M2. Workout 状态以中文字符串充当内部状态码

**证据**：

- `lib/control/workout_controller.dart:65,104,112,129,141,159,178,190,206,275,278,290,327,342`
- `lib/ui/pages/workout_page.dart:17-31`
- `lib/product/membership_status.dart:26`

WorkoutController 用中文字符串表达状态，UI 再通过 `_localizedWorkoutStatus()` 映射到 ARB。当前已知状态会本地化，中英文训练页测试也通过，因此这不是“英文用户必然看到中文”的现有功能故障；问题在于控制层与 UI 之间形成了脆弱的字符串契约，改文案可能静默破坏映射。

RevenueCat 的中文 `PurchaseFailedException.message` 不会直接显示，AccountController 会把异常类型映射成稳定错误码。测试模式按 UI V1 规范属于开发工具，不要求纳入产品页的完整 l10n。`membership_status.dart` 的 `'训练者'` fallback 则是一个独立的潜在漏翻译点。

**建议**：需要整理训练状态时，使用 enum 或稳定状态码，由 UI switch 到 ARB；单独处理 AppUser fallback。不要把这项写成当前系统性 i18n 失效。

#### M3. 分层规则与开发指南对资源常量位置互相矛盾

**证据**：

- `lib/platform/replay_utils.dart:20,27` 反向 import `../ui/app_theme.dart`
- `lib/ui/app_theme.dart:31-32` 定义 `modelPath`、`replayVideoName`
- `docs/development-guide.md:115` 明确要求这两个常量放在 `ui/app_theme.dart`

从依赖方向看，platform → UI 是反向边；但现行开发指南又明确把该位置写成项目约定。因此不能只把代码判为违反规范，真正问题是分层原则与具体维护规则不一致。

**建议**：先作一个小型架构决策：若 `config/` 是所有跨层资源常量的归属，就移动常量并同步开发指南与契约测试；若保留现状，则在分层文档中写明这是有意例外。不要只改一边。

### 🟢 轻微

#### L1. UI 保留了基本不可达的匿名头像回退算法

**证据**：

- `lib/ui/pages/leaderboard_page.dart:198-200,933-940`
- `lib/product/leaderboard_models.dart:115-120`
- `workers/membership-api/src/leaderboard.ts:264-266,384-392`

服务端返回 `anonymousAvatarKey`，Flutter JSON 模型也要求该字段存在且合法；正常页面路径仍在 UI 中保留一份相同 hash 回退。该函数基本不可达，并重复了服务端算法。

**建议**：删除 UI 回退函数，让已存在的 API/模型合同失败时明确报错；不建立跨 Dart/TypeScript 的共享抽象。

## 已核实的合理设计与非问题

- `PoseSilhouetteTracker` 只平滑视觉副本，不进入识别、准备门控或计数链路。`docs/plans/2026-07-12-pose-silhouette-design.md:10,30-49,96` 明确规定这一边界；纯 Dart 不等于必须放入 product，因此不建议移动。
- `ReplayControl` 虽小，但包含分支状态转换并有独立单测。是否继承 `ChangeNotifier` 不是 control 层成立条件，不建议仅因单一生产消费者而内联。
- `SignalExtractor` 的 `minConf` 和腕部检查阈值已有命名参数入口；语音 1-30 上限也由 `docs/modules/voice-themes.md` 与 `VoicePromptPlayer.playCount()` 明确定义，不作为魔法数问题。
- `lib/l10n/app_localizations*.dart` 是有意跟踪的生成文件，`docs/design/app-ui-v1.md` 已标明由 `flutter gen-l10n` 生成。手工添加文件头会被重新生成覆盖，`.gitignore` 注释也不提供保护。
- Workout 同步请求、返回结果与本地 `_replace` 都绑定捕获账号的 owner；账号切换后下一轮还有 `_isCurrent` 检查。现有代码不会把 B 的结果按 A owner 标记，因此不建议增加冗余的 await 后检查。
