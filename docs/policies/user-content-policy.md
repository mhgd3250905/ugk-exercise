# PushupAI 用户头像内容规则

版本：`2026-07-14`

生效范围：用户上传并可能在运动广场公开展示的自定义头像。训练相机画面仍只在设备端处理，不属于本规则所称的公开头像。

## 1. 上传与公开展示

- 用户必须在上传前主动勾选并接受当前版本规则；规则版本变化后需重新接受。
- 自定义头像优先于内置头像和 Google 账号头像展示。选择匿名身份时，不公开个人资料头像。
- 图片以 JPEG 保存于私有 R2 bucket；外部只能通过 Worker 返回仍处于有效公开状态的随机对象 URL。R2 不开放公共 bucket 访问。
- 头像会在运动广场向其他用户公开。请勿上传不希望公开的信息、他人肖像或可识别的敏感资料。

## 2. 禁止内容与行为

不得上传或利用头像传播：

- 裸露、色情或性剥削内容，尤其是涉及未成年人的内容；
- 血腥、暴力、威胁、自残鼓励或危险行为；
- 基于受保护特征的仇恨、贬损或骚扰；
- 违法内容、侵权内容或未经许可使用的他人肖像；
- 冒充他人、欺诈、误导性身份或规避处置的行为；
- 垃圾广告、恶意引流或其他扰乱服务的内容。

## 3. 举报、屏蔽与处置

- 登录用户可在非本人榜单行举报头像、举报用户或直接屏蔽用户；举报原因仅限裸露、暴力、仇恨、垃圾广告、冒充他人和其他违规。
- 举报成功会同时对举报者屏蔽目标，因此目标立即从举报者的榜单视图移除；这不会重写全局成绩或其他用户的名次。
- 举报进入人工审核队列。管理员可驳回举报、下架当前自定义头像、隐藏公开网络头像、暂停上传或恢复相关状态。
- 被举报头像已经替换时，旧举报标记为过期，不能据此删除新头像。所有管理动作写入审计记录。
- 重复或恶意举报、反复上传违规内容、规避限制等行为可导致上传权限暂停；严重违法内容可按适用法律处理。

## 4. 审核时限

- 运营人员至少每日检查一次开放举报队列。
- 涉及未成年人性内容、可信暴力威胁或其他紧急人身风险的举报，目标是在收到后 24 小时内先隐藏并复核。
- 其他举报目标在 3 个自然日内完成处置或记录继续调查的原因。
- 若无人能够持续履行上述检查，应暂停公开自定义头像上线，而不是带着无人审核的队列发布。

## 5. 保存、删除与申诉

- 当前头像保存至用户替换、主动删除、账号删除或管理员下架。替换和删除会先阻止旧版本继续公开，再尝试立即删除 R2 对象；失败对象保持不可公开并进入后续清理。
- Worker 的公开头像缓存最长 300 秒，因此下架后边缘缓存可能短暂存在。
- D1 保存头像所有权/状态、规则接受、举报、屏蔽和审核记录，不保存图片二进制。相关记录用于安全与审计，并在账号删除流程删除用户数据时按外键关系清理。
- 账号删除请求及处理时限见公开的[隐私政策与账号删除说明](https://pushupai-privacy.pages.dev/#account-deletion)。账号删除执行时必须先调用头像对象清理流程，再删除账号记录。
- 用户如认为头像被错误处置，可通过隐私政策公布的联系渠道请求复核，并提供账号和处置相关信息；不得发送密码、session token 或密钥。

## 6. 版本与发布要求

Worker 的权威版本常量为 `workers/membership-api/src/account.ts` 中的 `avatarPolicyVersion`。App、Worker 和本文件必须使用同一版本。

本规则在生产启用自定义头像前必须发布到公开可访问的政策页面，并同步复核 Google Play 的 Data safety、用户生成内容和内容分级声明。更新规则正文时必须更新版本常量并要求用户重新接受。
